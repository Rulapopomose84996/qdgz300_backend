# 项目整体实现结构与模块关系

## 1. 系统定位

QDGZ300 是三面阵雷达后端系统，目标是在受限硬件平台上持续输出可信、可操作的 `TrackData`。  
在 V1 形态下，系统按产品交付形态运行在单台后端服务器上，同时接入三阵面数据。

## 2. 业务边界

- 业务必达输出：`TrackData`
- 非业务必达但重要的中间产物：Plots、日志、遥测、录制数据
- 失败判据：无法持续输出可信 `TrackData`，或输出已失去实时性和可操作性

## 3. 模块划分

### 3.1 数据面

- `M01 Receiver`：UDP 收包、协议校验、分片重组、重排、投递
- `M02 SignalProc`：信号处理、GPU 路径、点迹生成
- `M03 DataProc`：点航关联、航迹维护、跨阵面融合
- `M04 Gateway`：对外会话、序列化、发送和输出节流

### 3.2 控制面

- `Orchestrator / Control`：状态机、事件裁决、命令桥接
- `Config`：参数管理与生效边界控制
- `TimeSync`：时间质量、守时和对外时间域映射
- `Health / Diagnostics / Logging / Recorder`：健康检查、诊断、观测与录制

## 4. 部署与拓扑约束

- 目标平台：麒麟 V10 ARM64
- 数据面与控制面按 NUMA / CPU 规划分离
- GPU 与数据面网卡优先位于同一 NUMA 节点
- 运行时数据、日志和抓包资产不与源码目录混用
