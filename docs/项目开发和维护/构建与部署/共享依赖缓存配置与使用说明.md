# 共享依赖缓存配置与使用说明

## 1. 目标

QDGZ300 默认采用“共享离线包目录 + 共享依赖缓存目录”模式，替代仓库内 `deps_offline/` 作为主流程依赖来源。

这样做的目的：

- 减少仓库体积
- 让 Windows、WSL、服务器三端复用同一套离线包
- 避免多个构建目录重复准备第三方依赖

## 2. 目录约定

### 2.1 Windows

- 项目源码：`D:\\WorkSpace\\Company\\Tower\\qdgz300_backend`
- 离线包目录：`D:\\WorkSpace\\ThirdPartyCache\\qdgz300_backend\\archives`

### 2.2 WSL

- 项目源码：`/mnt/d/WorkSpace/Company/Tower/qdgz300_backend`
- 离线包目录：`/mnt/d/WorkSpace/ThirdPartyCache/qdgz300_backend/archives`
- 交叉缓存目录：`/mnt/d/WorkSpace/ThirdPartyCache/qdgz300_backend/build/wsl-aarch64`

### 2.3 Linux 服务器

- 项目源码：`/home/devuser/WorkSpace/qdgz300_backend`
- 离线包目录：`/home/devuser/WorkSpace/ThirdPartyCache/qdgz300_backend/archives`
- 原生缓存目录：`/home/devuser/WorkSpace/ThirdPartyCache/qdgz300_backend/build/native-aarch64`

## 3. 对应脚本

- 服务器原生依赖准备：`scripts/prepare_native_deps.sh`
- WSL 交叉依赖准备：`scripts/prepare_wsl_cross_deps.sh`
- 服务器构建：`scripts/build/build_production.sh`
- WSL 构建：`scripts/build/build_wsl_cross.sh`

## 4. 环境变量

常用变量：

- `QDGZ300_OFFLINE_DEPS_DIR`
- `QDGZ300_DEPS_ROOT`
- `QDGZ300_DEPS_PREFIX`
- `QDGZ300_BUILD_TYPE`
- `QDGZ300_ENABLE_GPU`

## 5. 使用方式

### 5.1 WSL

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
bash scripts/build/build_wsl_cross.sh
```

### 5.2 Linux 服务器

```bash
cd /home/devuser/WorkSpace/qdgz300_backend
bash scripts/build/build_production.sh
```

## 6. 约束

- 共享缓存按平台隔离，不能混用 `native-aarch64` 与 `wsl-aarch64`。
- 构建脚本、依赖准备脚本和文档入口必须保持一致。
- 仓库内旧式离线流程只作为历史兜底材料，不作为主入口。
