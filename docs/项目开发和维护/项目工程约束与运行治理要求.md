# 项目工程约束与运行治理要求

## 1. 平台与工具链基线

- 目标系统：Kylin Linux Advanced Server V10, ARM64
- 构建系统：CMake 3.16.x
- 服务器 GPU 路径：CoreX 4.3.8 + clang 18.1.8
- WSL 交叉构建默认关闭 GPU

## 2. 数据面工程约束

- 数据面线程与管理面线程严格分离。
- 热路径禁止阻塞等待、隐式堆分配和不受控跨 NUMA 访问。
- 相邻阶段优先通过 SPSC 队列解耦。
- 队列满时采用可观测的丢弃策略，而不是阻塞等待。

## 3. 运行治理原则

- 顶层 `CMakeLists.txt` 只保留编排职责。
- 测试入口以脚本为准，不依赖人工拼装命令。
- 共享离线包目录与共享依赖缓存目录是默认依赖来源。
- 运行时目录统一收敛到 `data/` 或安装目录下的运行时子目录。

## 4. 参数治理

- 参数必须标注生效边界和变更级别。
- 控制面负责改参审计、版本化和冲突裁决。
- 运行中仅允许受控的动态变更，禁止绕过控制面直接改热路径行为。

## 5. 部署与升级治理

- 正式构建分为服务器原生构建和 WSL 交叉构建两条主线。
- 正式安装和服务操作统一走 `deploy/` 与 `scripts/ops/`。
- 版本升级、回归和运维检查需要以同一批脚本入口为准。

## 6. 文档治理

- 活跃文档只保留当前仍指导实现和运维的内容。
- 历史设计分卷、过程稿和 AI 材料统一归档。
- 冻结资产只做索引与命名规范，不改正文语义。
