# M01 本地压测与性能观察指南

> 适用场景：本地开发、WSL 验证、M01 接收链路的快速性能观察  
> 目标：统一原有 Dashboard 说明与本地压测操作步骤

## 1. 依赖对象

- `receiver_app`
- `fpga_emulator`
- `tools/receiver_perf_dashboard.py`

## 2. 准备构建

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
cmake -S . -B build_wsl_native_perf -DBUILD_TESTING=ON -DENABLE_GPU=OFF -DENABLE_PROTOBUF=OFF
cmake --build build_wsl_native_perf --target receiver_app fpga_emulator -j$(nproc)
```

## 3. 启动顺序

### 3.1 终端一：启动 Receiver

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
./build_wsl_native_perf/receiver_app --config config/receiver.yaml
```

### 3.2 终端二：启动性能面板

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
python3 tools/receiver_perf_dashboard.py \
  --metrics-url http://127.0.0.1:8080/metrics \
  --host 127.0.0.1 \
  --port 8090
```

浏览器访问：

```text
http://127.0.0.1:8090
```

### 3.3 终端三：启动 Sender 压测

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
./build_wsl_native_perf/fpga_emulator \
  --target-ip 127.0.0.1 \
  --target-port 9999 \
  --pps 400000 \
  --duration 120 \
  --payload-bytes 992 \
  --threads 2 \
  --output build_wsl_native_perf/runtime_logs/sender_run.json
```

## 4. 观察重点

- 包率与吞吐量是否稳定
- 丢包曲线是否持续抬升
- CPU 曲线是否长期顶满
- `receiver_packets_received_total` 等指标是否随压测持续增长

## 5. 快速排障

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
pgrep -a receiver_app || true
pgrep -a fpga_emulator || true
curl -s http://127.0.0.1:8080/metrics | egrep "^receiver_packets_received_total|^receiver_bytes_received_total" || true
```

## 6. 退出

优先使用各终端中的 `Ctrl + C`。若需要统一结束：

```bash
cd /mnt/d/WorkSpace/Company/Tower/qdgz300_backend
pkill -f receiver_app || true
pkill -f receiver_perf_dashboard.py || true
pkill -f fpga_emulator || true
```
