# 算法第一环设计冻结项清单

## 1. 用途

本文把“算法第一环”相关的前置设计冻结项从过程性问答稿转写为正式专题文档，用于说明在进入实质算法研究或实现前，哪些口径必须先闭合。

## 2. 已暴露的关键不一致

### 2.1 第一环边界不统一

- 系统工程口径常把“接收、重组、解交织”归入 M01。
- 算法研究口径常把“距离向匹配滤波 / 脉压”视为第一环。
- 如果边界不先冻结，输入、输出和验收对象都无法唯一确定。

### 2.2 时序参数口径未闭合

- `PRI / PRF / CPI / N` 的关系在现有材料中存在冲突或歧义。
- 这会直接影响多普勒刻度、速度模糊区和相干处理口径。

### 2.3 采样与输入格式未完全冻结

- `Fs`、`DataType`、`ChannelMask`、I/Q 排列、极化表达方式仍存在“研究假设”和“协议显式字段”不完全一致的问题。
- 这会影响算法输入张量、参考信号生成和量纲解释。

### 2.4 分片与样点规则未完全闭环

- `SampleCountFixed`
- `SampleCount`
- `ShortCodeSampleCount`
- `LongCodeSampleCount`

这些量的关系若不冻结，第一环输入长度和补零后可解释性都不稳定。

### 2.5 Snapshot 与尾包丢失场景存在缺口

- Snapshot 仅在尾包携带。
- 协议又允许超时补零输出不完整帧。
- 因此算法侧可能拿到“可输出的帧”却拿不到对应 Snapshot，当前需明确处理策略。

## 3. 本期必须补齐的冻结项

1. 文档仲裁优先级：协议、冻结集成表、固件偏差说明、抓包、研究报告、代码之间谁是最终真值源。
2. 第一环边界定义：到底研究“重组/解交织”“脉压”还是两者一起。
3. 本期有效波形与时序表：波形类型、`Fs / B / PulseWidth / PRI / CPI / N`。
4. RAW 数据格式冻结表：`DataType`、`ChannelMask`、字节序、通道顺序、极化表达、缩放方式。
5. 样点与分片规则闭环说明：固定样点数、长短码样点数与分片长度如何对应。
6. 第一环验收标准：输入输出定义、golden model、误差容限、质量标记、时延和吞吐预算。
7. 真实数据与基准结果：控制面输入、原始数据、Snapshot、golden reference。
8. 固件与协议偏差清单：现网实现与协议 V3.1 的差异。

## 4. 使用建议

- 若需求是做工程实现，先对照 `docs/设计/实现设计/` 和 `docs/冻结资产/`。
- 若需求是继续算法研究，先完成本清单中的冻结项闭合，再进入算子推导和实现设计。
- 历史问答稿已转入归档，仅保留为追溯材料。
