# 项目骨架与目录约定

> 性质：规划视图。  
> 用途：说明仓库期望的工程化组织方式，而不是直接等同于当前代码实况。

## 1. 顶层目录职责

```text
qdgz300_backend/
├── cmake/        # 工具链、查找模块、编译选项
├── config/       # 运行配置
├── data/         # 运行期数据落点
├── deploy/       # 安装与服务资产
├── docs/         # 文档体系
├── include/      # 公共头与模块接口
├── scripts/      # 构建、测试、运维脚本
├── src/          # 模块实现
├── tests/        # 单元与集成测试
└── tools/        # 压测与辅助工具
```

## 2. 目录治理规则

- 顶层 `CMakeLists.txt` 只做模块编排。
- 模块源码和编译定义尽量下沉到各自子目录。
- 测试、部署、工具和运行时目录分离，不混入业务源码树。
- 文档中的“规划视图”与“代码实况”必须分开描述。

## 3. 当前模块分层

- `M01`：接收与协议处理
- `M02`：信号处理
- `M03`：数据处理与航迹
- `M04`：网关与对外输出
- `control / orchestrator / config / time / health / diagnostics / recorder`：控制面与基础设施
