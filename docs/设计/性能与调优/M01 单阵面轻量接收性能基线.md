# M01 单阵面轻量接收性能基线

更新时间：2026-03-08

## 目的

记录当前 `bench_m01_rx_limit` 在 `kds` 服务器上的单阵面轻量级接收能力基线，作为后续“完整数据面处理”优化前的对照样本。

这里的“轻量级接收”指：

- `UdpReceiver`
- `PacketParser`
- `Validator`

不包含：

- `Dispatcher`
- `Reassembler`
- `Reorderer`
- `DeliveryInterface`

## 测试环境

- 主机：`kds`
- 测试日期：2026-03-08
- 拓扑：
  - 接收口：`enP1s25f0 = 192.168.1.101`
  - 发送口：`enP1s25f3` 移入独立 network namespace，配置 `192.168.1.102/24`
- 链路：双万兆口本机真实自回环
- 接收程序：`build_release/tools/benchmarks/bench_m01_rx_limit`
- 发送程序：`build_release/fpga_emulator`
- 包长：约 `1024B` 总包长
  - `payload_bytes = 992`
  - 协议头约 `32B`
- 发送线程数：`2`

## 方法

- 使用 `bench_m01_rx_limit` 作为接收端。
- 使用 `fpga_emulator` 从 `txns` namespace 经 `enP1s25f3` 向 `192.168.1.101:9999` 发包。
- 先做短时阶梯搜索，再做窄区间复测，最后做 `60s` 确认。
- “稳定”按连续复测中接收 PPS 与发送 PPS 基本一致、无明显失稳定义。

## 结果摘要

### 阶梯搜索

| 目标 PPS | 实际 TX PPS | 实际 RX PPS | 结论 |
|---|---:|---:|---|
| 250000 | 249995.99 | 250001.68 | 稳定 |
| 300000 | 300025.93 | 299975.13 | 稳定 |
| 325000 | 325040.09 | 325105.28 | 稳定 |
| 350000 | 350012.44 | 314423.65 | 明显失稳 |
| 400000 | 399993.82 | 322078.04 | 明显失稳 |

### 临界区复测

| 目标 PPS | 现象 |
|---|---|
| 328000 | 有时接近满收，有时出现明显波动，不稳定 |
| 329000 | 多次复测出现明显抖动，不稳定 |
| 330000 | 持续存在掉速，不稳定 |
| 325000 | 连续复测可维持稳定 |

### 最终确认

- `325000 PPS`
- `60s` 确认结果：
  - `TX = 325043.81 PPS`
  - `RX = 325045.01 PPS`
  - gap `-0.014%`

## 结论

当前代码在“轻量级接收路径”下，单阵面可复现的最大稳定能力约为：

- `325k PPS`

这组结果只能代表：

- 网卡收包
- 协议头解析
- 基础合法性校验

不能代表真实 `receiver_app` 的完整业务处理能力。

## 对比说明

同日对真实 `receiver_app` 单阵面完整路径做过基线测试，`1500B` 标准帧下无丢包基线仅约：

- `85k PPS`
- `1.02 Gbps`

这说明当前主要性能损失发生在完整数据面后半段，而不是轻量接收前半段。
