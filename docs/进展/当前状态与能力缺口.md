# 当前状态与能力缺口

## 1. 当前状态

- 文档结构已收口为 `进展 / 规划 / 基线 / 设计 / 冻结资产 / _archive` 六层。
- 工程主线已形成固定入口：
  - 服务器原生构建：`scripts/build/build_production.sh`
  - WSL 交叉构建：`scripts/build/build_wsl_cross.sh`
  - 单元测试：`scripts/test/test_unit.sh`
  - 集成测试：`scripts/test/test_integration.sh`
- 运行时约束已明确：
  - 数据面与管理面按 NUMA / CPU 规划隔离。
  - 共享离线包目录与共享依赖缓存目录是默认依赖来源。
  - `data/` 作为运行时数据统一落点。

## 2. 当前已形成闭环的部分

- 麒麟 V10 ARM64 服务器原生构建入口已固化。
- WSL 交叉构建入口已固化。
- 安装与 systemd 运维脚本链路已具备正式入口。
- M01 接收模块已具备可运行、可调试、可压测的真实实现。
- 性能基线、部署基线、冻结资产和设计主线已完成基础归档与入口清理。

## 3. 主要能力缺口

### 3.1 模块能力

- M02 仍以 CPU fallback / 最小可测路径为主，真实 GPU 算子链尚未恢复完整生产行为。
- M03、M04、Control 具备最小可测实现，但离最终业务版本仍有差距。
- M04 真实 protobuf、真实 TCP 会话管理、真实 UDP 发送与组播参数仍需继续收口。

### 3.2 模块接口与数据契约

- M01 与 M02 间 `RawBlock` 结构、所有权模型和队列类型仍需统一。
- `RawBlock` 2MB payload 上限是否完全覆盖真实 FPGA CPI 规模，仍待硬件联调验证。
- 算法第一环边界、输入格式、验收口径已形成专题清单，但仍需由设计冻结资料进一步闭合。

### 3.3 运维与回归

- 服务部署与压测指南已固化，但回归检查仍依赖人工执行。
- 调优清单与性能基线已沉淀，后续需要与真实业务链路继续对齐。

## 4. 近期优先级

1. 继续收口 M02、M04、Control 的真实业务能力。
2. 统一 M01 到 M02 的数据投递与内存所有权模型。
3. 用真实协议、真实数据和真实压测场景补齐算法与网关验证闭环。
4. 继续保持文档入口、脚本入口和真实代码行为一致。
