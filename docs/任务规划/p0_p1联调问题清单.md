# P0 P1 联调问题清单

记录日期：2026-03-11

1. `include/qdgz300/m01_receiver/delivery/raw_block.h` 中的 `RawBlock` 使用内嵌 `uint8_t payload[2MB]`，而 `include/qdgz300/common/types.h` 中供 M02 使用的 `RawBlock` 采用 `uint8_t *payload` 指针语义。后续需要统一结构定义或提供正式适配层。
2. M01 `RawBlockAdapter` 输出 `SPSCQueue<std::shared_ptr<RawBlock>, 64>`，M02 `GpuDispatcher` 消费 `SPSCQueue<RawBlock *, 64>`。当前联调测试通过桥接层维持 shared ownership，但正式链路仍需统一所有权模型。
3. M01 固定 2MB `RawBlock` payload 是否覆盖真实 FPGA CPI 数据量仍未实测验证。该容量需要在 P2 硬件联调阶段结合真实采样规模确认。
