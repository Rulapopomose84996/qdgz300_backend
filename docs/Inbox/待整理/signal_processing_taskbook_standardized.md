## 1.2 系统概述

### 系统或子系统的用途

信号数据综合处理机分为信号处理软件、数采系统和时序产生电路，为轻型多功能雷达的信号处理、数据处理和时序产生的核心部分。信号处理软件运行于信号处理软件，主要完成接收系统送来的中频回波信号A/D采样、数字下变频、脉冲压缩、目标检测、单脉冲测角、数字DBF、干扰侦察等功能，并将处理结果送给数采系统。

### 软件在系统中的作用

- 对前10路数字下变频后的数据可完成常规DBF或者自适应DBF，形成方位△与Σ数据；
- 对DBF形成的方位Σ与△数据进行数字脉压、MTD和CFAR（仅对Σ）；
- 根据CFAR检测结果结合Σ与△计算目标的角误差信息，并送给数采系统；
- 配合总站完成T/R检查、接收机校准、内/外配相功能；
- 具备故障检测、故障定位及接口通讯检查功能，并能送出检测结果给数采系统。

### 数字下变频（DDC）

获取采样数据后，需进行数字下变频处理，具体的处理方法如下：

设第i个通道A/D采样后的数据为：

```text
f_i(n)=A(n) cos(2πf_0 n + 0.5μn^2 + φ_i(n))
```

式中：

f0=125MHz为中频信号载频；

μ——线性调频斜率。

I、Q正交分解操作方法：

```text
I_{f_i}(n) = (f_i(n) × cos(2πf_0 n)) ⨂ LF
```

```text
Q_{f_i}(n) = (f_i(n) × sin(2πf_0 n)) ⨂ LF
```

式中：

⨂——卷积运算；

LF——低通滤波器。

在本系统中，LF设计为40阶的加Kaiser窗的滤波器，40阶滤波器系数(归一化后)为：

```text
LF=[0.0000   -0.0001   -0.0001    0.0002    0.0007   -0.0000   -0.0024 0.0025   0.0039    0.0095    -0.0000   -0.0201   -0.0174    0.0241 0.0535   -0.0000   -0.1024    -0.0918   0.1462    0.4904    0.6557 0.4904   0.1462    -0.0918    -0.1024   -0.0000   0.0535    0.0241 -0.0174   -0.0201   -0.0000    0.0095   0.0039    -0.0025   -0.0024 -0.0000   0.0007    0.0002    -0.0001   -0.0001   -0.0000]。
```

注：在软件实现低通滤波卷积操作时，设信号的长度为L，卷积后应取最中心的L个滤波点，可避免卷积运算产生的附加延迟。

### 搜索（关键功能）

#### 数字脉冲压缩(DPC)

对4个通道数据需进行DPC运算。DPC有时域、频域两种实现方式，在本系统中我们采用频域实现方式，具体做法如下：

选择要进行FFT的点数N；

计算匹配滤波器：

```text
MS1 = exp(j × π × (B/τ × t^2))
```

式中：

B——工作带宽，B=25e6MHz；

τ——当前回波的脉冲宽度；

t——时间，计算方法为 t=0:τ/L:(τ-τ/L)；

L——匹配滤波器长度，L=τ×f_{s0}，f_{s0}=25MHz。

计算得到上述结果后：

```text
MS2 = conj(fliplr(MS1))
```

`fliplr` 表示把滤波器按照倒序排列，`conj` 表示 `MS1` 的数据先倒序后再虚部取反。

使用MATLAB计算L点汉明窗滤波器 `Ham`；

计算加窗后的频域匹配滤波器 `MS_FFT = FFT(MS2 × Ham)`，此处FFT运算的点数为 a) 步中的 N 点；

对4个通道数据补零，长度变为N，再进行N点的FFT变换，结果记为 `S_FFT`；

获得频域脉压系数 `S_FFT_DPC = S_FFT × MS_FFT`；

进行N点的IFFT运算，获得时域脉压系数 `S_DPC = IFFT(S_FFT_DPC)`。

注：在实际的软件实现中，根据脉冲宽度、点数N分别计算出不同的匹配滤波器系数MS_FFT并存储，根据实际需要在脉压时直接调用，不需要每次计算。

#### 稀疏处理

用滑窗的方法遍历所有需要做恒虚警率检测的数据。计算滑窗内数据模值，有若干点大于规定门限则判定有干扰，将其清零。

#### 动目标指示(MTI)

读取数据包头MTI功能开关控制字，如控制字为“开”，则进行MTI计算，本雷达MTI的设计采用三脉冲对消，对和通道进行脉压后的数据进行MTI杂波对消；如控制字为“关”，则跳过此过程。具体的对消方法如下：

选择当前状态下的信号积累次数np；

设某路信号在第 i 次积累的信号记为 `S_i(n)`，i=1,…,np，n=1,…,RN（距离通道个数），MTI 对消方法为：

`S_{j-MTI}(n) = S_i(n) - 2S_{i-1}(n) + S_{i-2}(n)`，其中 j=i-2，i=3,…,np。

注：在MTI对消前，当前CPI数据总量为RN(距离通道)×np(积累次数)个，在完成对消后变为RN(距离通道)×(np-2)(积累次数)。

#### 动目标检测(MTD)

对所有通道经过MTI处理后，对和通道进行MTD处理，具体的做法为：

将经过 MTI 对消后的 np-2 次信号 `S_{j-MTI}(n)` 在距离通道上对齐（未做 MIT 则为 np 次）；

对同一距离通道对应的np-2个信号依次进行FFT处理，之后再后移下一个距离通道进行同样处理：

`S_MTI_MTD(n) = FFT(S_{1-MTI}(n), ..., S_{np-2-MTI}(n))`，距离通道 n=1,…,RN。

注：在MTD处理后数据总量为RN(距离通道)×(np-2)(速度通道)。

#### 恒虚警率(CFAR)检测与目标合并

在和通道完成MTD运算之后，距离通道与速度通道构成矩阵数据进行CFAR检测，CFAR检测也只处理Σ路数据。具体步骤如下：

对所有数据实部与虚部平方和后开方取模，即 `|S_MTI_MTD(n)|` 操作，获得 `S_M` 数据；

取所有速度通道 nv 的距离通道方向的前32点（总共为 nv×32 点数据）与后32点 `S_M` 数据求平均，比较取二者的最大值，即为噪声电平 `σ`。此时获得门限 `Sh1 = Kσ`，门限系数 K 由数采系统发送过来，将 Sh1 与数采系统发送的门限值 Sh2 进行比较，取最大的值为噪声门限；

按照图4所示，选择取大(Go-CFAR)方式进行CFAR检测，先检测第一个速度通道的所有距离通道，以此类推，直至完成所有速度通道的检测；

设数采系统送来的目标速度通道范围为：[Vns1, Vne1]，杂波抑制舍弃的速度通道为Vnum个，才此时对应的速度通道范围为[Vnum+1, Vn0- Vnum-1]，求[Vns1, Vne1]、[Vnum+1, Vn0- Vnum-1]之间的交集作为最后的杂波抑制速度通道输出范围：[Vns0, Vne0]；根据数采系统发送来的工作方式控制字，选择[Vns1, Vne1]是否起作用，若起作用则对第c)步中检测出的所有目标进行筛选，满足该速度通道范围的目标予以保留；

> 配图占位：此处为“搜索状态CFAR检测原理图”。

设雷达系统的目标容量为On，按照第d)步目标筛选后的结果，对目标的幅度按照从大到小排序，取出前On个目标，记录这些目标的距离通道Rn、速度通道Vn，即为目标合并结果。

#### 旁瓣匿影

读取数据包头旁瓣匿影功能开关控制字，如控制字为“开”，在进行4.1.1.8步骤b）时，将辅助通道（MTD后数据）对应的距离通道目标幅度值乘以主旁瓣幅度比值（由目标处理软件送来），记为门限值Sh3，与门限值Sh2进行比较，取最大的值为噪声门限，记为新的Sh2；如控制字为“关”，则跳过此步骤。

#### 杂波图计算

读取数据包头杂波图功能开关控制字，如控制字为“开”，进行杂波图计算，计算方法如下：

建立杂波图，按照短码和长码分为2个数组，数组按距离、方位、俯仰三个参数进行标记；

对MTD合路结果单个距离单元去掉剔除通道后计算均值，得到距离维杂波信息，并拆分为长码杂波图和短码杂波图（仅在发射状态下处理）；

CFAR完成后，对当前波位检测得到的目标信息进行剔除，长短目标分别处理，在各自杂波图中查找对应（按距离、方位、俯仰三维信息进行匹配）的杂波底噪，计算目标幅度与底噪的比值，设置合适阈值进行剔除，可增加保护区域，该区域不进行杂波目标剔除计算（如临近威力范围及左右边界）；

杂波图数据每个波位进行更新，设置更新开关，可在一段时间后关闭更新。

在进行4.1.1.8步骤b）时，在杂波图中查找对应距离单元的门限值，记为Sh4与门限值Sh2进行比较，取最大的值为噪声门限；如控制字为“关”，则跳过此步骤。

#### 目标参数获取

##### 距离测量

目标处理软件送来的距离通道范围为[Rns0, Rne0]。根据目标处理软件发送来的工作方式控制字，选择 [Rns0, Rne0]是否起作用，若起作用则对检测出的所有目标进行筛选，满足该距离通道范围的目标予以保留。

采用幅度加权法，以回波幅度为权重，加权回波的时延。方法如下：在合并为同一目标的多个目标点中搜索具有相同多普勒通道号且幅度最大的几个（最少2个），设为 `p_1(τ_1, f, y_1)` 和 `p_2(τ_2, f, y_2)`，采用幅度加权法计算内插值如下：

```text
R_i = ((y_1 × τ_1 + y_2 × τ_2) / (y_1 + y_2)) C_1 + C_2
```

式中C1为考虑光速和量化单位后的系数，单位0.01m；C2为校正值。对于3个点的情况，可用下式计算：

```text
R_i = ((y_1 × τ_1 + y_2 × τ_2 + y_3 × τ_3) / (y_1 + y_2 + y_3)) C_1 + C_2
```

##### 速度测量

选择幅度最大的多普勒通道号。

##### 角度测量

在和通道进行CFAR检测后的目标数据中，记下所有目标到的距离通道，在MTI后的方位差和俯仰差数据中，分别找到对应的距离通道及±2个通道数据，形成最大10路信号进行脉压、幅相校准和自适应DBF权值相乘后，形成 (np-2)×10 维矩阵，对该矩阵进行MTD计算，设计算后方位差、俯仰差支路目标包络最大点对应的值分别为 fεΔmax 和 fβΔmax，和支路目标包络最大点对应的值为 fΣmax，再按下述公式求方位角误差 `coe_{βΔ}` 和俯仰角误差 `coe_{εΔ}`，量化单位 0.001：

```text
coe_{βΔ} = |f_{βΔmax} / f_{Σmax}| × cos(φ_{βΔ} - φ_{Σ})
coe_{εΔ} = |f_{εΔmax} / f_{Σmax}| × cos(φ_{εΔ} - φ_{Σ})
```

#### 目标数据打包

根据CFAR检测出的第i个目标的距离通道Rin和速度通道Vin，取出Σ波束、方位差Δβ波束和俯仰差Δε波束MTD后的该位置沿距离通道方向5个点的I、Q数据，并使目标点(Rin, Vin)在中心。将每个目标的(Rin, Vin)、5个点的I、Q数据、噪声电平（占用数据表AGC幅度值位置）、本波位目标的相关状态等信息打包后通过Rapid I/O交换机全部送给目标处理软件。

### 通讯检查

信号处理软件读取目标处理软件送来的状态控制要求，判定为“通讯检查”，读取目标处理软件送来的测试数据包，将测试数据包原数回送给目标处理软件，误码率优于10-6。

### 幅相校准

根据目标处理软件送来的状态控制要求，信号处理进行幅相计算时，信号处理响应领示脉冲信号产生时序(如图5所示)作为其采样门，完成对25μs的领示信号采样。

> 配图占位：此处为“幅相校准时序安排”。

幅相校准的处理过程如图3所示，具体的计算过程如下：

信号处理软件接收到采样后的4路25MHz领示信号，信号处理完成对这4路信号的数字下变频；

对下变频后的每一路数据进行256点的FFT运算，并对FFT后的数据进行求模运算。对FFT后的I、Q数据、求模后的数据均进行保存；

找到每一路的模值最大点位置 m0，及其前后的各两点，共5点数据记为 `|d^i|_{m0-j}`，i=1，2，…，12 表示子阵编号，j=-2,-1,0,1,2 表示最大模值附近的5点数据。同时取出对应位置的 I、Q 数据，记为 `d^i_{m0-j}`。以第1路接收子阵作为参考，计算4路通道之间的幅相不一致性：幅度不一致性：`Ki=[1，(1/5)∑_j |d^1|_{m0-j}/|d^i|_{m0-j}，…，(1/5)∑_j |d^1|_{m0-j}/|d^i|_{m0-j}，…]`；相位不一致性：`Δθi=[0，(1/5)∑_j(φ(d^1_{m0-j})-φ(d^2_{m0-j}))，…，(1/5)∑_j(φ(d^1_{m0-j})-φ(d^i_{m0-j}))，…]`。

将计算的结果按通道编号标记，打包发送给目标处理软件。

> 配图占位：此处为“幅相校准软件处理流程”。

### T/R检查

#### T/R收态检查

根据数采系统送来的状态控制要求，信号处理软件进行T/R检查计算。其中采样、数字下变频处理方式不变。信号处理过程同搜索功能，对Σ路进行目标检测，若有目标，送出目标的幅度、速度通道号、距离和相位到目标处理，若没有目标，信号处理软件送无目标标识。

#### T/R发态检查

根据数采系统送来的状态控制要求，信号处理软件进行T/R检查计算。其中采样、数字下变频处理方式不变。信号处理过程同搜索功能，选取辅助通道作为Σ路，若有目标，送出目标的幅度、速度通道号、距离和相位到目标处理，若没有目标，信号处理软件送无目标标识。

### 方向图检查

根据数采系统送来的状态控制要求，信号处理软件进行方向图检查计算。其中采样、数字下变频处理方式不变。信号处理过程同搜索功能，对Σ路进行目标检测，若有目标，送出目标的幅度、速度通道号、距离和相位到目标处理，若没有目标，信号处理软件送无目标标识。

### 回波显示

根据数采系统送来的状态控制要求，信号处理软件配合实现回波显示。其中采样、数字下变频处理方式不变。信号处理过程同搜索功能，对Σ路进行目标检测，若有目标，按照状态字中要求对应送出相应通道的目标数据到目标处理，若没有目标，信号处理软件送无目标标识。

### 干扰侦察

对输入10路数据进行DBF合成处理，对形成的方位Σ和方位Δ两路数据进行2048点的FFT运算后求模并计算所有采样点的模值平均值，将每一点的模值与数采系统送来的干扰门限进行对比，记录过门限点数，统计连续四点过门限的干扰个数N，0＜N＜5，判断干扰类型为多音干扰；N≥5，判断干扰类型为单音干扰；若有干扰连续过门限点数大于100判断干扰类型为宽带干扰。

## 4.5 接口

DSP0/1一组处理奇数CPI数据，其中DSP0通过SRIO接口从FPGA接收DDC后数据进行DBF和DPC处理，然后通过HYPERLINK接口发送给DSP1，DSP1对数据进行MTI等后续处理将目标结果通过EMIF接口送给数采系统。

DSP2/3一组处理偶数CPI数据，其中DSP2对应DSP0，DSP3对应DSP1，处理流程同上。

### EMIF接口（EMIF）

EMIF（External Memory Interface）接口为TI DSP自带的外部存储扩展接口。该接口用于接收数采系统发送过来的工作命令和参数并将检测结果发送回数采系统。

输入数据参见《XX雷达软件数据表》中的表A.2；

输出数据参见《XX雷达软件数据表》中的表A.6~A.10。

### DDR3接口（DDR3）

DDR3接口用于DSP读写DDR3存储芯片，用来缓存处理过程中的数据。

### SRIO接口（SRIO）

SRIO接口用于FPGA和DSP之间的通讯，FPGA将DDC处理后的结果通过该接口发送给DSP1和DSP3。

### HYPERLINK接口（HYPERLINK）

HYPERLINK接口用于DSP0/2和DSP1/3间的通讯，速度20Gbps，传输数据量为每个PRF的脉压结果。
