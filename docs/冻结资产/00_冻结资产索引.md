# 冻结资产索引

## 1. 目录职责

`docs/冻结资产/` 用于保存已经冻结、需要长期引用、但不应在日常整理中改写正文语义的资产。

本目录允许：

- 增加索引说明
- 规范文件命名
- 调整引用入口

本目录不允许：

- 改写协议正文含义
- 在未仲裁的情况下重述冻结事实

## 3. 与活跃文档的关系

- 当前可执行的工程约束，以 `docs/基线/` 和 `docs/设计/使用指南/` 为准。
- 当前系统设计主线，以 `docs/设计/系统架构/` 为准。
- 当活跃文档与冻结资产发生冲突时，应先核对冲突是否属于“活跃解释”与“冻结原文”之间的层次差异；若确有事实冲突，应回到冻结资产和真实实现重新仲裁。
