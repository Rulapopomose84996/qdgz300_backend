# 波位控制与 UDP 数据包格式说明

## 1. 文档目的

本文档用于整理当前通信接口调试阶段的波位控制表与 UDP 数据包格式约定，形成结构清晰、内容一致、便于实现与联调的 Markdown 版本说明。

本文档仅基于现有约定内容整理，不额外引入新的协议语义。

---

## 2. 波位控制表

### 2.1 基本定义

波位控制表暂定长度为 **2048 字节**。

### 2.2 字段约定

- **包头**：`0x55AAFF00`
- **CPI 计数**：2 字节
- **其余内容**：先填充 `0`

### 2.3 说明

当前阶段，波位控制表结构较为简单，主要用于标识波位控制信息的起始以及携带 CPI 计数。除上述有效字段外，其余字节统一按 `0` 填充，便于后续扩展和联调。

---

## 3. UDP 包格式

### 3.1 基本定义

单个 UDP 包暂定长度为 **5120 字节**。

一个 PRT 需要发送：

- `4 × 4 = 16` 个 UDP 包

即：
- 每个 PRT 的数据按 4 组发送；
- 每组 4 个 UDP 包；
- 合计 16 个 UDP 包。

---

## 4. UDP 包内容组成

UDP 包由以下两部分组成：

- 包信息
- 包数据

---

## 5. 包信息

### 5.1 包头

- **包头**：`0x55AAFF03`

### 5.2 CPI 计数

- **长度**：2 字节
- **范围**：`0 ~ 65535`

### 5.3 PRT 计数

- **长度**：2 字节

### 5.4 通道计数

- **长度**：1 字节
- **取值定义**：
  - `0`：和
  - `1`：俯仰差
  - `2`：方位差
  - `3`：匿隐通道

### 5.5 包计数

- **长度**：1 字节
- **范围**：`0 ~ 8`

> 注：原始记录中给出的范围为 `0-8`，此处保持一致，不做额外修订。

### 5.6 IQ 点数

- **长度**：2 字节

### 5.7 包尾

包尾规则如下：

- 只有**当前波位的最后一个 PRT 的最后一个 UDP 包**携带有效包尾：`0x55AAFF30`
- 其余 UDP 包的包尾统一填充 **4 字节 `0`**

### 5.8 包信息字段汇总

| 字段名 | 长度 | 说明 |
|---|---:|---|
| 包头 | 4 字节 | 固定为 `0x55AAFF03` |
| CPI 计数 | 2 字节 | 范围 `0~65535` |
| PRT 计数 | 2 字节 | 当前 PRT 序号/计数 |
| 通道计数 | 1 字节 | `0/1/2/3` 分别对应和、俯仰差、方位差、匿隐通道 |
| 包计数 | 1 字节 | 范围 `0~8` |
| IQ 点数 | 2 字节 | 当前包中有效 IQ 点数 |
| 包尾 | 4 字节 | 仅最后一个特定 UDP 包为 `0x55AAFF30`，其他包填 `0` |

---

## 6. 包数据

### 6.1 数据区大小

单个 UDP 包总长度为 **5120 字节**。

其中包信息占用 **16 字节**，因此数据区可用长度为：

`5120 - 16 = 5104 字节`

即：

- **数据区域可用长度**：`5104 字节`

### 6.2 数据组织方式

一个 PRT 的 16 个 UDP 包按通道顺序组织：

- 第 1 个 4 包：和路数据
- 第 2 个 4 包：俯仰数据
- 第 3 个 4 包：方位数据
- 最后 1 个 4 包：匿隐数据

也就是说，每个通道对应 4 个 UDP 包，4 个通道共计 16 个 UDP 包。

### 6.3 单通道 4 包的数据承载示例

示例约定如下：

- 前 3 包 UDP 包各自包含 **1276 个 IQ**
- 第 4 包 UDP 包包含 **912 个 IQ**
- 剩余空间全部填充 `0`

该示例用于说明一个通道内 4 个 UDP 包的数据装载方式。

## 7. UDP 包字节布局示意

根据原始表格内容，UDP 包前部字段布局如下：

| 字节偏移 | 字段 |
|---|---|
| 0~3 | `0x55AAFF03` |
| 4~5 | CPI 计数 |
| 6~7 | PRT 计数 |
| 8 | 通道计数 |
| 9 | 包计数 |
| 10~11 | IQ 点数 |
| 12~15 | 包尾 |
| 16~17 | I |
| 18~19 | Q |

### 7.1 说明

- 字节 `0~15` 为包信息区。
- 字节 `16` 开始为 IQ 数据区。
- IQ 数据按 `I`、`Q` 顺序排列。

## 8. 实现约束汇总

为便于开发与联调，现将关键约束统一整理如下：

### 8.1 波位控制表约束

1. 波位控制表长度固定为 **2048 字节**。
2. 表头固定为 `0x55AAFF00`。
3. CPI 计数字段长度为 **2 字节**。
4. 除已定义字段外，其余内容全部填 `0`。

### 8.2 UDP 包约束

1. 单个 UDP 包长度固定为 **5120 字节**。
2. 一个 PRT 固定发送 **16 个 UDP 包**。
3. 每个通道对应 **4 个 UDP 包**。
4. 4 个通道顺序依次为：
   - 和路
   - 俯仰差
   - 方位差
   - 匿隐通道
5. 包头固定为 `0x55AAFF03`。
6. CPI 计数长度为 **2 字节**，范围 `0~65535`。
7. PRT 计数长度为 **2 字节**。
8. 通道计数长度为 **1 字节**，取值 `0~3`。
9. 包计数长度为 **1 字节**，范围按当前约定为 `0~8`。
10. IQ 点数字段长度为 **2 字节**。
11. 仅当前波位最后一个 PRT 的最后一个 UDP 包使用包尾 `0x55AAFF30`。
12. 其他 UDP 包在包尾位置均填充 **4 字节 `0`**。
13. 数据区有效长度固定为 **5104 字节**。
14. IQ 数据从字节偏移 `16` 开始存放，按 `I`、`Q` 顺序组织。
