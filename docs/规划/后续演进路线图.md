# 后续演进路线图

## 1. 目标

后续规划聚焦于把当前“可构建、可部署、部分可测”的工程形态，继续收口为“业务链路可闭环、接口口径一致、可持续回归”的长期维护形态。

## 2. 近期路线

### 2.1 数据链路收口

- 统一 M01 到 M02 的 `RawBlock` 契约、所有权模型和队列类型。
- 用真实 FPGA CPI 规模验证 payload 上限、补零策略和重组边界。

### 2.2 核心模块恢复

- M02：补真实 GPU 路径、Pinned Memory、Stream / Event 生命周期。
- M03：继续扩展真实航迹处理与跨阵面融合逻辑。
- M04：补真实 protobuf、会话管理、发送与节流策略。
- Control：补真实模块健康来源、状态裁决和命令桥接。

### 2.3 联调与验证

- 继续以服务器原生构建作为权威验证路径。
- 用模块级测试、集成测试和标准压测组合形成回归入口。
- 将算法专题清单逐项冻结为可执行设计输入。

## 3. 中期目标

- 打通真实接收、真实信号处理、真实航迹输出的最小业务闭环。
- 完成关键冻结参数的协议侧、实现侧、压测侧三方对齐。
- 建立稳定的回归和诊断资料沉淀方式，减少过程稿扩散。
