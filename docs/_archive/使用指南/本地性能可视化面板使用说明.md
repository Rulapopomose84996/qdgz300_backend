# 本地性能可视化面板使用说明

本文档说明如何在本地使用 `tools/receiver_perf_dashboard.py` 对雷达接收模块进行实时性能观察。

## 1. 功能

- 每秒自动刷新
- 显示 4 条实时曲线：
  - 吞吐量（Mbps）
  - 包率（pps）
  - CPU 使用率（%）
  - 丢包速率（count/s）
- 默认保留最近 300 秒历史
- 无数据库、无外部服务依赖

## 2. 启动前提

1. Receiver 已运行，并暴露 Prometheus 指标端点（默认 `http://127.0.0.1:8080/metrics`）。
2. 本机有 Python3。

## 3. 启动命令

在项目根目录执行：

```bash
python3 tools/receiver_perf_dashboard.py \
  --metrics-url http://127.0.0.1:8080/metrics \
  --host 127.0.0.1 \
  --port 8090
```

浏览器打开：

```text
http://127.0.0.1:8090
```

## 4. 常用参数

- `--interval-sec`：采样周期（默认 1 秒）
- `--history-sec`：历史窗口长度（默认 300 秒）
- `--packet-size-bytes`：吞吐量显示使用的包长（默认 1024）
- `--receiver-pid`：指定 Receiver PID，用于 CPU 曲线
- `--receiver-proc-name`：按进程名自动查找 PID（默认 `receiver_app`）

示例：

```bash
python3 tools/receiver_perf_dashboard.py \
  --metrics-url http://127.0.0.1:8080/metrics \
  --port 8090 \
  --interval-sec 1 \
  --history-sec 600 \
  --receiver-proc-name receiver_app
```

## 5. 与压测联动建议

推荐先启动 dashboard，再启动发送压测与 Receiver：

1. 启动 Receiver
2. 启动 dashboard
3. 启动 Sender 压测
4. 在页面观察曲线变化趋势

## 6. 注意事项

- 若页面显示“指标抓取异常”，先检查 `--metrics-url` 是否可访问。
- 若 CPU 曲线为空，通常是 PID 未找到，可手动传 `--receiver-pid`。
- 该工具用于开发阶段压测观察，不用于生产级监控。
